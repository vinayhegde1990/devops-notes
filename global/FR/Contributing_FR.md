# Contributions

## Comment contribuer à ce projet ?

Veuillez utiliser l’un des éléments ci-dessous pour clarifier vos modifications avant de les effectuer.

### Fonctionnalité

- Vérifiez d’abord s’il existe des problèmes ouverts qui répondent déjà aux exigences de fonctionnalités dont vous disposez.
- Sinon, utilisez le [modèle de demande de fonctionnalité](.github/ISSUE_TEMPLATE/feature_request.md) avec un titre et une description claire des champs qu'il contient.

### Bogue

- Vérifiez d’abord s’il existe des problèmes ouverts qui traitent déjà des bugs que vous avez rencontrés.
- Sinon, utilisez le [modèle de rapport de bogue](.github/ISSUE_TEMPLATE/bug_report.md) avec un titre et une description claire des champs qu'il contient.

#### Créer une Pull Request

1. Forkez le dépôt et créez votre branche à partir de « master ».
1. Effectuez les modifications souhaitées dans les dossiers et/ou sous-dossiers **langue/technologie** appropriés → ajouter → valider → envoyer vos fichiers vers la branche.
1. Si la **langue/technologie** de votre section n'existe pas, n'hésitez pas à créer une nouvelle entrée dans la table des matières.
1. Créez la [Pull Request via votre support préféré](https://docs.github.com/en/pull-requests/collaborating-with-pull-requests/proposing-changes-to-your-work-with-pull-requests/creating-a-pull-request)
1. Effectuez une demande d'extraction **séparée** pour une fonctionnalité ou un bug. Ne les combinez pas.
